# HR Operations Copilot

人事労務担当者の **人件費シミュレーション・給与チェック・過去事例検索** を支援するSaaSダッシュボードのプロトタイプです。

> 📌 **コンセプト**: 人事SaaSは入力ツールではなく **判断支援ツール** である

## 🖥️ 画面構成

### 1. ダッシュボード (`/`)
人事データの全体状況を把握するための概要画面
- 人件費サマリー（KPIカード）
- 社員数推移チャート
- 人件費予測（実績と予測）
- AIアラート

### 2. 人件費シミュレーション (`/simulation`)
条件を変更して人件費の影響をシミュレーション
- 昇給率設定
- 採用人数設定
- 社会保険料率変更
- シミュレーション結果のグラフ表示
- AI分析・提案

### 3. 給与チェック (`/payroll`)
先月との差分を確認し、異常値を検知
- 先月との差分一覧
- 異常値検知
- 会社ルールチェック
- コメントログ

### 4. 過去事例検索 (`/cases`)
過去の対応事例や関連規程を検索
- 検索バー
- 類似事例表示
- 関連規程
- 対応履歴

### 5. AI Copilot (`/copilot`)
人事担当の判断を支援するAIチャット
- AIチャットインターフェース
- データ根拠表示
- クイックアクション
- レポート生成

## 🎨 デザインスタイル

- **配色**: 白・グレー・ブルーアクセント
- **スタイル**: シンプル、カードUI
- **参考**: Linear, Stripe Dashboard, Vercel Dashboard

## 🛠️ 技術スタック

- **Framework**: React 18
- **Language**: TypeScript
- **Styling**: TailwindCSS
- **Icons**: Lucide React
- **Charts**: Recharts
- **Build Tool**: Vite

## 🚀 セットアップ

### 現在のペインポイント
1. 人件費予算作成がExcel手作業
2. 給与チェックが会社固有ルール込みで難しい
3. 組織変更でHRデータの連続性が失われる

5. 人事システムが分析やシミュレーションに弱い

## ✨ UX特徴

- 🔍 **異常検知**: 給与の異常値を自動検知
- 📊 **シミュレーション**: 条件変更によるリアルタイム計算
- 🤖 **AI提案**: データに基づく判断支援
- 📈 **差分表示**: 前月との比較を視覚化
